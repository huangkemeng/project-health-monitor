# 监控项分组功能需求文档 (PRD)

## 1. 功能概述

### 1.1 产品愿景
为 Health Monitor 项目监控平台增加监控项分组管理功能，帮助用户更好地组织和分类监控项，提高管理效率。

### 1.2 目标用户
- SMB 销售团队的技术负责人
- 需要管理大量监控项的系统管理员
- 需要按项目/环境/业务线分类监控的用户

### 1.3 核心目标
- 支持创建、编辑、删除监控分组
- 支持将监控项分配到指定分组
- 支持按分组查看和筛选监控项
- 支持分组的批量操作

## 2. 详细功能描述

### 2.1 分组管理功能

#### 2.1.1 创建分组
- **功能描述**: 用户可以创建新的监控分组
- **输入字段**:
  - 分组名称 (必填): 1-50字符，支持中文、英文、数字、空格
  - 分组描述 (可选): 0-200字符
  - 分组颜色 (可选): 预设10种颜色供选择，用于视觉区分
- **业务规则**:
  - 分组名称在同一用户下唯一
  - 默认创建"未分组"系统分组，不可删除
  - 最多支持创建50个分组
- **错误处理**:
  - 名称重复提示:"分组名称已存在"
  - 超出数量限制提示:"最多只能创建50个分组"

#### 2.1.2 编辑分组
- **功能描述**: 修改分组的基本信息
- **可编辑字段**: 名称、描述、颜色
- **业务规则**:
  - 系统分组"未分组"不可编辑
  - 修改名称时需校验唯一性
- **错误处理**:
  - 名称重复提示:"分组名称已存在"

#### 2.1.3 删除分组
- **功能描述**: 删除指定分组
- **业务规则**:
  - 系统分组"未分组"不可删除
  - 删除分组时，该分组下的监控项自动移动到"未分组"
  - 需要二次确认
- **确认提示**: "确定要删除分组"XXX"吗？该分组下的X个监控项将被移动到未分组。"

#### 2.1.4 查看分组列表
- **功能描述**: 展示所有分组及统计信息
- **展示字段**:
  - 分组名称
  - 分组颜色标识
  - 监控项数量
  - 正常/警告/异常监控项数量统计
  - 创建时间
  - 更新时间
- **排序规则**: 按更新时间倒序，系统分组"未分组"始终置顶
- **交互功能**:
  - 点击分组名称查看该分组下的监控项
  - 支持拖拽排序（可选，V2版本）

### 2.2 监控项与分组关联功能

#### 2.2.1 创建监控项时选择分组
- **功能描述**: 在创建监控项时指定所属分组
- **界面元素**:
  - 下拉选择框，显示所有分组列表
  - 默认选中"未分组"
  - 支持快速创建新分组（可选）
- **业务规则**:
  - 不选择时默认归入"未分组"

#### 2.2.2 编辑监控项时修改分组
- **功能描述**: 修改监控项所属分组
- **界面元素**: 下拉选择框，显示所有分组
- **业务规则**:
  - 可随时更改分组
  - 更改后实时监控列表更新

#### 2.2.3 批量移动监控项到分组
- **功能描述**: 选择多个监控项，批量移动到指定分组
- **操作流程**:
  1. 在监控列表中选择多个监控项（复选框）
  2. 点击"移动到分组"按钮
  3. 选择目标分组
  4. 确认移动
- **业务规则**:
  - 支持跨页选择（可选，V2版本）
  - 最多一次移动100个监控项
- **成功提示**: "成功将X个监控项移动到分组XXX"

### 2.3 分组筛选与展示功能

#### 2.3.1 监控列表按分组筛选
- **功能描述**: 在监控列表页面按分组筛选显示
- **界面元素**:
  - 左侧分组导航栏（可折叠）
  - 显示分组名称和监控项数量
  - 点击分组筛选该分组下的监控项
  - "全部"选项显示所有监控项
- **视觉设计**:
  - 当前选中分组高亮显示
  - 分组颜色标识显示在名称前

#### 2.3.2 分组统计看板
- **功能描述**: 在仪表盘展示分组维度的统计
- **展示内容**:
  - 各分组的监控项总数
  各分组的健康状态分布（饼图/柱状图）
  - 最近24小时各分组的告警次数
- **交互功能**:
  - 点击分组跳转到该分组的监控列表

### 2.4 分组权限控制（可选，V2版本）

#### 2.4.1 分组级别的权限管理
- **功能描述**: 支持将分组分享给其他用户或团队
- **权限类型**:
  - 查看权限: 只能查看分组和监控项
  - 编辑权限: 可以修改监控项配置
  - 管理权限: 可以添加/删除监控项，修改分组信息

## 4. 数据结构建议

### 4.1 数据库表设计

#### 4.1.1 分组表 (monitor_groups)

```sql
CREATE TABLE monitor_groups (
    id VARCHAR(36) PRIMARY KEY,
    user_id VARCHAR(36) NOT NULL,
    name VARCHAR(50) NOT NULL,
    description VARCHAR(200),
    color VARCHAR(7) DEFAULT '#3B82F6', -- 十六进制颜色值
    sort_order INT DEFAULT 0,
    is_default BOOLEAN DEFAULT FALSE, -- 系统默认分组"未分组"
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    UNIQUE KEY uk_user_group_name (user_id, name)
);

-- 索引
CREATE INDEX idx_monitor_groups_user_id ON monitor_groups(user_id);
CREATE INDEX idx_monitor_groups_sort_order ON monitor_groups(sort_order);
```

#### 4.1.2 监控项表添加分组字段

```sql
-- 在 monitors 表中添加 group_id 字段
ALTER TABLE monitors ADD COLUMN group_id VARCHAR(36) NULL;

-- 添加外键约束
ALTER TABLE monitors ADD FOREIGN KEY (group_id) REFERENCES monitor_groups(id) ON DELETE SET NULL;

-- 创建索引
CREATE INDEX idx_monitors_group_id ON monitors(group_id);
```

### 4.2 API 接口设计

#### 4.2.1 分组管理接口

**创建分组**
```
POST /api/groups
Request:
{
  "name": "生产环境",
  "description": "生产环境API监控",
  "color": "#EF4444"
}

Response:
{
  "id": "uuid",
  "name": "生产环境",
  "description": "生产环境API监控",
  "color": "#EF4444",
  "monitor_count": 0,
  "created_at": "2024-01-01T00:00:00Z",
  "updated_at": "2024-01-01T00:00:00Z"
}
```

**获取分组列表**
```
GET /api/groups

Response:
{
  "items": [
    {
      "id": "uuid",
      "name": "未分组",
      "description": "",
      "color": "#6B7280",
      "is_default": true,
      "monitor_count": 5,
      "health_summary": {
        "normal": 3,
        "warning": 1,
        "critical": 1
      },
      "created_at": "2024-01-01T00:00:00Z",
      "updated_at": "2024-01-01T00:00:00Z"
    }
  ],
  "total": 1
}
```

**更新分组**
```
PUT /api/groups/:id
Request:
{
  "name": "生产环境API",
  "description": "更新后的描述",
  "color": "#10B981"
}

Response: 更新后的分组对象
```

**删除分组**
```
DELETE /api/groups/:id
Response: 204 No Content
```

#### 4.2.2 监控项与分组关联接口

**批量移动监控项到分组**
```
POST /api/groups/:id/monitors
Request:
{
  "monitor_ids": ["uuid1", "uuid2", "uuid3"]
}

Response:
{
  "moved_count": 3
}
```

**修改监控项分组**
```
PUT /api/monitors/:id/group
Request:
{
  "group_id": "uuid"  // null 表示移动到未分组
}

Response: 更新后的监控项对象
```

**按分组获取监控项列表**
```
GET /api/groups/:id/monitors?page=1&limit=20

Response:
{
  "items": [...],
  "total": 10,
  "page": 1,
  "limit": 20
}
```

## 6. 非功能性需求

### 6.1 性能要求
- 分组列表加载时间 < 500ms
- 分组筛选监控项 < 1s
- 批量移动100个监控项 < 3s

### 6.2 安全要求
- 用户只能操作自己的分组
- 分组名称需要防止 XSS 攻击
- 删除分组需要二次确认

### 6.3 兼容性要求
- 支持移动端查看分组列表
- 分组颜色在深色模式下正常显示

## 8. 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 现有监控项数据迁移 | 高 | 创建"未分组"并自动关联所有现有监控项 |
| 分组删除后监控项归属 | 中 | 删除时自动移动到"未分组" |
| 分组数量过多影响性能 | 低 | 限制最大分组数为50 |
