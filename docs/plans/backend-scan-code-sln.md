### 一、逻辑与结构错误
| Bug 类型 | 典型示例 | AI 检测依据 |
|---------|----------|--------------|
| **死循环** | `while (i < 10) { /* 忘记 i++ */ }` | 循环变量未更新、退出条件恒真 |
| **条件永远为真/假** | `if (a == 1 && a == 2)` | 约束求解、数据流分析 |
| **分支遗漏** | `switch (day) { case 1: ... break; // 缺少 default 或其它 case` | 枚举不完整、逻辑覆盖 |
| **错用运算符** | `if (a = b)` (赋值当比较) | 语法+语义检查 |
| **错误的短路求值** | `if (obj != null && obj.method())` 反过来顺序 | 空指针风险 |

### 二、运行时异常与资源管理
| Bug 类型 | 典型示例 | AI 检测依据 |
|---------|----------|--------------|
| **空指针/引用解引用** | `String s = null; s.length();` | 路径分析、@NotNull 注解 |
| **资源未释放** | 打开文件/连接后未 close（不放在 finally 或 try-with-resource） | 配对检查（open/close） |
| **数组越界** | `int[] arr = new int[5]; arr[5] = 1;` | 索引范围分析 |
| **类转换异常** | `Object o = "hello"; Integer i = (Integer) o;` | 类型流分析 |
| **除零错误** | `int avg = total / count;` 若 count 可能为 0 | 数值约束 |
| **栈溢出风险** | 无限递归 或 超大局部数组 | 调用图深度/内存布局 |

### 三、并发与多线程问题
| Bug 类型 | 典型示例 | AI 检测依据 |
|---------|----------|--------------|
| **竞态条件** | 非原子性操作（`if(map.containsKey(k)) map.put(k,v)`） | 缺乏同步的读写 |
| **死锁** | 循环加锁顺序不一致 | 锁顺序分析 |
| **内存可见性** | 缺少 volatile / 同步导致线程看不到更新 | 缺少 happens-before |
| **误用线程安全类** | 多个线程操作 HashMap 而非 ConcurrentHashMap | 类型+并发注解 |
| **线程泄漏** | 无限创建线程（不使用池） | 创建/销毁模式 |

### 四、性能与效率问题（算法低效的细化）
| Bug 类型 | 典型示例 | AI 检测依据 |
|---------|----------|--------------|
| **循环内重复计算** | `for(...){ size = list.size(); }` 应提到外面 | 循环不变代码外提 |
| **使用 O(N²) 而非 O(N)** | 循环内使用 `indexOf` 而不用哈希集 | 复杂度启发式规则 |
| **字符串拼接在循环** | `s += i;` 循环内 -> 改用 StringBuilder | 字符串操作模式 |
| **频繁装箱拆箱** | `sum += i` (Integer 对象) | 类型推断 |
| **未使用懒加载** | 每次都创建大对象 | 初始化时机检查 |
| **数据库 N+1 查询** | 循环内逐条查 SQL | ORM 日志分析（需结合运行时） |

### 五、异常处理与健壮性
| Bug 类型 | 典型示例 | AI 检测依据 |
|---------|----------|--------------|
| **吞掉异常** | `try { ... } catch (Exception e) { }` | 空 catch 块 |
| **捕获过于宽泛** | `catch (Exception e)` 并按业务处理 | 异常层次分析 |
| **finally 中抛异常** | 掩盖原始异常 | 控制流分析 |
| **未闭合资源** | 普通 `InputStream` 不 close | 资源规范检查 |
| **错误码被忽略** | `int ret = func();` 不检查 ret | 未使用的返回值 |

### 六、安全漏洞（AI 擅长模式匹配）
| Bug 类型 | 典型示例 | AI 检测依据 |
|---------|----------|--------------|
| **SQL 注入** | `"SELECT * FROM user WHERE name='" + name + "'"` | 字符串拼接 SQL |
| **XSS** | 直接将 request 参数输出到 HTML 未转义 | 污点分析 |
| **硬编码凭证** | `password = "123456"` | 正则/敏感词 |
| **不安全的反序列化** | `ObjectInputStream.readObject()` 无过滤 | 方法调用识别 |
| **路径遍历** | `new File("/var/data/" + userInput)` | 检查 `../` 等 |
| **日志注入** | 用户输入直接写日志 | 换行符未过滤 |

### 七、代码风格与可维护性（虽非 Bug，但影响质量）
| 类型 | 示例 | AI 检测依据 |
|-----|------|--------------|
| **重复代码** | 三段类似的复制粘贴代码 | 相似代码块检测 |
| **过长方法/类** | 超 200 行或 50 个方法 | 度量阈值 |
| **未使用的变量/导入** | `int unused = 0;` | 数据流死存储 |
| **魔法数字** | `if (status == 3)` 应该用常量 | 字面量检查 |
| **不一致命名** | `getUser()` 和 `fetch_user()` | 命名风格规则 |

### 八、其他常见可检测 Bug
| Bug 类型 | 典型示例 | AI 检测依据 |
|---------|----------|--------------|
| **浮点数比较错误** | `if (a == 0.1)` 而实际是 0.1000000001 | 精度警告 |
| **时间处理错误** | `new Date(2023-1900, 11, 32)` 无效日期 | 日期范围 |
| **时区缺失** | 使用 `Calendar.getInstance()` 而不指定时区 | 默认时区依赖 |
| **字符串编码错误** | `new String(bytes, "UTF-8")` 未 catch 异常 | 未处理受检异常 |
| **正则表达式灾难性回溯** | `(a+)+b` 匹配大量 a | 正则复杂度分析 |
