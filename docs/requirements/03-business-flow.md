### 3.3.3 状态转换规则

| 当前状态 | 操作 | 目标状态 | 触发条件 | 业务规则 |
|:-------:|:----:|:-------:|---------|---------|
| - | 创建 | active | 用户成功创建监控项 | 立即开始探测 |
| active | 暂停 | paused | 用户点击暂停按钮 | 停止探测，保留配置 |
| paused | 恢复 | active | 用户点击恢复按钮 | 重新开始探测 |
| active | 删除 | archived | 用户确认删除 | 停止探测，软删除 |
| paused | 删除 | archived | 用户确认删除 | 软删除 |

### 3.4.3 健康状态判定规则

```python
# 伪代码：健康状态判定逻辑

def determine_health_status(monitor, consecutive_failures, last_response_time, last_http_code):
    """
    判定监控项的健康状态
    
    参数:
        monitor: 监控项配置
        consecutive_failures: 连续失败次数
        last_response_time: 上次响应时间(ms)
        last_http_code: 上次HTTP状态码
    
    返回:
        health_status: normal/warning/critical
        alert_level: None/warning/critical
    """
    
    # 如果连续失败次数未达到阈值，保持normal状态
    if consecutive_failures < monitor.retry_times:
        return "normal", None
    
    # 连续失败达到阈值，判定异常级别
    is_slow = last_response_time >= monitor.warning_threshold
    is_server_error = last_http_code and 500 <= last_http_code < 600
    
    if is_slow or is_server_error:
        return "critical", "critical"
    else:
        return "warning", "warning"


def on_probe_success(monitor):
    """探测成功时的处理"""
    if monitor.health_status != "normal":
        # 状态从异常恢复
        previous_status = monitor.health_status
        monitor.health_status = "normal"
        monitor.consecutive_failures = 0
        
        # 发送恢复通知
        send_recovery_notification(monitor, previous_status)
    else:
        monitor.consecutive_failures = 0


def on_probe_failure(monitor, error_info):
    """探测失败时的处理"""
    monitor.consecutive_failures += 1
    
    # 检查是否达到告警阈值
    if monitor.consecutive_failures >= monitor.retry_times:
        new_status, alert_level = determine_health_status(
            monitor,
            monitor.consecutive_failures,
            error_info.response_time,
            error_info.http_code
        )
        
        # 状态变化或持续异常时触发告警
        if monitor.health_status != new_status:
            monitor.health_status = new_status
            trigger_alert(monitor, alert_level)
        elif not is_in_silence_period(monitor):
            # 静默期结束后再次告警
            trigger_alert(monitor, alert_level)
```

### 3.7.2 数据生命周期

| 数据类型 | 创建时机 | 更新时机 | 删除时机 | 保留期限 |
|---------|---------|---------|---------|---------|
| 用户账号 | 注册时 | 修改信息时 | 注销时 | 永久 |
| 监控配置 | 创建监控时 | 编辑监控时 | 删除监控时 | 软删除后90天 |
| Webhook配置 | 创建时 | 编辑时 | 删除时 | 软删除后90天 |
| 探测记录 | 每次探测后 | 不更新 | 自动清理 | 30天 |
| 告警记录 | 触发告警时 | 恢复时更新 | 自动清理 | 90天 |

## 3.8 异常处理流程

### 3.8.1 探测异常处理

| 异常类型 | 处理策略 | 记录方式 | 是否触发告警 |
|---------|---------|---------|:-----------:|
| 网络超时 | 标记失败，响应时间=超时值 | error_msg="timeout" | 是 |
| DNS解析失败 | 标记失败 | error_msg="dns_error" | 是 |
| 连接被拒绝 | 标记失败 | error_msg="connection_refused" | 是 |
| SSL证书错误 | 标记失败 | error_msg="ssl_error" | 是 |
| 5xx服务器错误 | 标记失败，记录状态码 | http_code=5xx | 是 |
| 4xx客户端错误 | 标记失败，记录状态码 | http_code=4xx | 是 |
| 状态码不匹配 | 标记失败 | error_msg="status_mismatch" | 是 |
| 响应时间超时 | 标记失败 | error_msg="response_timeout" | 是 |
