> **文档版本**: v1.0  
> **最后更新**: 2026-05-02

## 4.1 文档概述

### 4.1.1 文档目的

本文档详细描述项目健康监控系统的所有功能需求，包括功能清单、详细需求规格、验收标准和界面原型说明，是开发团队进行系统设计和编码的主要依据。

### 4.1.2 文档范围

本文档涵盖以下功能模块：
- 用户认证模块
- 监控项管理模块
- Webhook配置模块
- 监控执行引擎
- 告警管理模块
- 状态展示模块
- 历史记录模块

### 4.1.3 需求标识规则

本文档采用以下需求标识规则：

```
格式: FR-[模块代号]-[序号]

模块代号:
- AUTH: 用户认证模块 (Authentication)
- MON: 监控项管理模块 (Monitor)
- WH: Webhook配置模块 (Webhook)
- ENG: 监控执行引擎 (Engine)
- ALERT: 告警管理模块 (Alert)
- DASH: 状态展示模块 (Dashboard)
- HIST: 历史记录模块 (History)

示例:
- FR-AUTH-001: 用户注册功能
- FR-MON-001: 创建监控项功能
```

## 4.3 用户认证模块

### 4.3.1 FR-AUTH-001: 用户注册

#### 需求描述

允许新用户注册系统账号，创建个人账户。

#### 功能规格

| 属性 | 描述 |
|-----|------|
| **功能名称** | 用户注册 |
| **需求编号** | FR-AUTH-001 |
| **优先级** | Must |
| **参与者** | 访客用户 |
| **前置条件** | 用户未登录 |
| **触发条件** | 用户点击注册按钮 |

#### 输入数据

| 字段名 | 类型 | 必填 | 校验规则 | 说明 |
|-------|------|:----:|---------|------|
| username | String | 是 | 3-20字符，字母开头，只允许字母数字下划线 | 用户名，唯一 |
| email | String | 是 | 有效邮箱格式 | 邮箱地址，唯一 |
| password | String | 是 | 8-32字符，必须包含字母和数字 | 登录密码 |
| confirm_password | String | 是 | 必须与password一致 | 确认密码 |

#### 验收标准

```gherkin
场景: 用户成功注册
  Given 用户访问注册页面
  When 用户输入有效的用户名"zhangsan"
  And 用户输入有效的邮箱"zhangsan@example.com"
  And 用户输入密码"Password123"
  And 用户输入确认密码"Password123"
  And 用户点击注册按钮
  Then 系统创建用户账号
  And 返回注册成功消息
  And 自动跳转到登录页面

场景: 用户名已存在
  Given 系统中已存在用户名为"zhangsan"的用户
  When 新用户尝试使用"zhangsan"注册
  Then 系统返回错误"用户名已被使用"
  And 用户保留在注册页面

场景: 密码强度不足
  Given 用户访问注册页面
  When 用户输入密码"12345678"
  Then 系统返回错误"密码必须包含字母和数字"
```

#### 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 用户名已存在 | 返回错误码400，message="用户名已被使用" |
| 邮箱已存在 | 返回错误码400，message="邮箱已被注册" |
| 密码格式不符 | 返回错误码400，message="密码必须包含字母和数字，长度8-32位" |
| 两次密码不一致 | 返回错误码400，message="两次输入的密码不一致" |
| 数据库连接失败 | 返回错误码500，message="服务器内部错误" |

#### 处理流程

```

│   │                                                          │
│   ▼                                                          │
│  提交登录请求                                                 │
│   │                                                          │
│   ▼                                                          │
│  查询用户信息                                                 │
│   │                                                          │
│   ├── 用户不存在 ──▶ 返回错误: "用户名或密码错误"              │
```

#### 输出结果

| 结果 | 说明 |
|-----|------|
| 成功 | 返回JWT Token和用户信息，跳转监控大盘 |
| 失败 | 显示"用户名或密码错误"，不提示具体哪个错误 |

#### Token规格

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 604800,
  "user": {
    "id": "uuid",
    "username": "zhangsan",
    "email": "zhangsan@example.com",
    "created_at": "2026-05-02T10:00:00Z"
  }
}
```

Token Claims:
- `sub`: 用户ID
- `username`: 用户名
- `iat`: 签发时间
- `exp`: 过期时间（7天）

#### 验收标准

```gherkin
场景: 用户成功登录
  Given 用户已注册账号
  When 用户输入正确的用户名和密码
  And 用户点击登录按钮
  Then 系统验证凭据成功
  And 返回JWT Token
  And 跳转到监控大盘

场景: 用户名不存在
  Given 用户输入不存在的用户名
  When 用户点击登录按钮
  Then 系统返回错误"用户名或密码错误"
  And 不提示具体是用户名还是密码错误

场景: 密码错误
  Given 用户输入正确的用户名
  And 用户输入错误的密码
  When 用户点击登录按钮
  Then 系统返回错误"用户名或密码错误"
  And 记录失败次数
```

#### 安全机制

| 机制 | 规则 |
|-----|------|
| 密码错误锁定 | 连续5次密码错误后，锁定账号15分钟 |
| 错误信息模糊 | 不区分"用户名不存在"和"密码错误" |
| Token过期 | 7天后自动过期，需要重新登录 |
| Token刷新 | 支持使用refresh_token刷新（可选） |

### 4.3.3 FR-AUTH-003: 用户登出

#### 需求描述

允许已登录用户退出系统，清除登录状态。

#### 功能规格

| 属性 | 描述 |
|-----|------|
| **功能名称** | 用户登出 |
| **需求编号** | FR-AUTH-003 |
| **优先级** | Must |
| **参与者** | 已登录用户 |
| **前置条件** | 用户已登录 |
| **触发条件** | 用户点击登出按钮 |

1. 用户点击登出按钮
2. 客户端清除本地存储的Token
3. 调用服务端登出接口（可选，用于记录登出日志）
4. 跳转登录页面

#### 验收标准

```gherkin
场景: 用户成功登出
  Given 用户已登录系统
  When 用户点击登出按钮
  Then 客户端清除Token
  And 跳转登录页面
  And 再次访问需要登录的页面时要求重新登录
```

### 4.4.2 FR-MON-002: 编辑监控项

#### 需求描述

允许用户修改已创建的监控项配置。

#### 功能规格

| 属性 | 描述 |
|-----|------|
| **功能名称** | 编辑监控项 |
| **需求编号** | FR-MON-002 |
| **优先级** | Must |
| **参与者** | 监控项所有者 |
| **前置条件** | 用户已登录，拥有该监控项 |
| **触发条件** | 用户点击"编辑"按钮 |

#### 输入数据

同创建监控项，增加字段：

| 字段名 | 类型 | 必填 | 说明 |
|-------|------|:----:|------|
| id | UUID | 是 | 监控项ID |

#### 业务规则

1. 只能编辑自己的监控项
2. 编辑后重置连续失败计数
3. 修改interval后，重新调度探测任务
4. 修改webhook_id后，后续告警使用新的Webhook

#### 验收标准


## 4.6 监控执行引擎

### 4.6.1 FR-ENG-001: 定时探测执行

#### 需求描述

系统按配置的间隔定时执行监控探测任务。

#### 功能规格

| 属性 | 描述 |
|-----|------|
| **功能名称** | 定时探测执行 |
| **需求编号** | FR-ENG-001 |
| **优先级** | Must |

#### 技术要求

| 项目 | 规格 |
|-----|------|
| 调度方式 | 基于时间轮的定时任务调度 |
| 并发数 | 最大100个并发探测 |
| 精度 | 秒级精度，允许±5秒偏差 |
| 容错 | 单任务失败不影响其他任务 |

#### 调度算法

```python
# 简化版调度逻辑
class ProbeScheduler:
    def __init__(self):
        self.task_queue = PriorityQueue()  # 按执行时间排序
    
    def add_monitor(self, monitor):
        """添加监控项到调度队列"""
        next_run = now() + monitor.interval
        self.task_queue.put((next_run, monitor.id))
    
    def run(self):
        while True:
            next_run, monitor_id = self.task_queue.get()
            sleep_until(next_run)
            
            # 异步执行探测
            asyncio.create_task(self.execute_probe(monitor_id))
            
            # 重新加入队列
            monitor = get_monitor(monitor_id)
            if monitor and monitor.status == 'active':
                next_run = now() + monitor.interval
                self.task_queue.put((next_run, monitor_id))
```

### 4.6.2 FR-ENG-002: HTTP探测

#### 需求描述

执行HTTP/HTTPS请求，检测目标服务的可用性。

#### 功能规格

| 属性 | 描述 |
|-----|------|
| **功能名称** | HTTP探测 |
| **需求编号** | FR-ENG-002 |
| **优先级** | Must |

#### 探测逻辑

```python
async def probe_http(monitor):
    start_time = time.time()
    
    try:
        async with aiohttp.ClientSession() as session:
            async with session.request(
                method=monitor.method,
                url=monitor.url,
                headers=monitor.headers,
                data=monitor.body if monitor.method in ['POST', 'PUT'] else None,
                timeout=aiohttp.ClientTimeout(total=monitor.timeout),
                ssl=False if monitor.url.startswith('http://') else True
            ) as response:
                
                response_time = int((time.time() - start_time) * 1000)
                
                # 检查状态码
                if response.status == monitor.expected_status:
                    return ProbeResult(
                        status='success',
                        http_code=response.status,
                        response_time=response_time
                    )
                else:
                    return ProbeResult(
                        status='failure',
                        http_code=response.status,
                        response_time=response_time,
                        error_msg=f'状态码不匹配，期望{monitor.expected_status}，实际{response.status}'
                    )
                    
    except asyncio.TimeoutError:
        return ProbeResult(
            status='failure',
            response_time=monitor.timeout * 1000,
            error_msg='请求超时'
        )
    except aiohttp.ClientError as e:
        return ProbeResult(
            status='failure',
            error_msg=f'请求失败: {str(e)}'
        )
```
