## 9.1 测试策略

### 9.1.1 测试层级

```
┌─────────────────────────────────────────────────────────────┐
│                        测试金字塔                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│                      ┌─────────┐                            │
│                      │  E2E    │  10%                       │
│                      │  测试   │  端到端测试                 │
│                      └────┬────┘                            │
│                   ┌───────┴───────┐                         │
│                   │               │                         │
│                ┌─────────┐   ┌─────────┐                    │
│                │ 集成测试 │   │ API测试 │  20%               │
│                │         │   │         │  接口集成           │
│                └────┬────┘   └────┬────┘                    │
│             ┌───────┴───────────┐                           │
│             │                   │                           │
│          ┌─────────┐      ┌─────────┐                       │
│          │ 单元测试 │      │ 单元测试 │  70%                 │
│          │ (服务层)│      │ (工具类)│  函数/方法            │
│          └─────────┘      └─────────┘                       │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 9.1.2 测试类型

| 测试类型 | 目标 | 工具 | 覆盖率目标 |
|---------|------|------|:---------:|
| 单元测试 | 验证函数/方法正确性 | pytest | 80%+ |
| 集成测试 | 验证模块间协作 | pytest | 关键路径 |
| API测试 | 验证接口契约 | pytest + requests | 100%接口 |
| E2E测试 | 验证完整业务流程 | Playwright | 核心流程 |

## 9.2 测试用例

### 9.2.1 用户认证模块

#### TC-AUTH-001: 用户注册

| 用例ID | TC-AUTH-001 |
|-------|-------------|
| 用例名称 | 用户成功注册 |
| 前置条件 | 用户未注册 |
| 测试步骤 | 1. 访问注册页<br>2. 输入有效用户名、邮箱、密码<br>3. 点击注册 |
| 预期结果 | 注册成功，跳转登录页 |
| 优先级 | P0 |

#### TC-AUTH-002: 用户登录

| 用例ID | TC-AUTH-002 |
|-------|-------------|
| 用例名称 | 用户成功登录 |
| 前置条件 | 用户已注册 |
| 测试步骤 | 1. 访问登录页<br>2. 输入正确的用户名和密码<br>3. 点击登录 |
| 预期结果 | 登录成功，获取Token，跳转大盘 |
| 优先级 | P0 |

### 9.2.2 监控项模块

#### TC-MON-001: 创建监控项

| 用例ID | TC-MON-001 |
|-------|-------------|
| 用例名称 | 成功创建监控项 |
| 前置条件 | 用户已登录，已配置Webhook |
| 测试步骤 | 1. 点击新建监控<br>2. 填写名称和URL<br>3. 选择Webhook<br>4. 点击保存 |
| 预期结果 | 监控项创建成功，状态为active，开始探测 |
| 优先级 | P0 |

#### TC-MON-002: 探测执行

| 用例ID | TC-MON-002 |
|-------|-------------|
| 用例名称 | 定时探测正常执行 |
| 前置条件 | 存在active状态的监控项 |
| 测试步骤 | 等待探测间隔时间 |
| 预期结果 | 产生探测记录，更新最后探测时间 |
| 优先级 | P0 |

### 9.2.3 告警模块

#### TC-ALERT-001: 告警触发

| 用例ID | TC-ALERT-001 |
|-------|-------------|
| 用例名称 | 连续失败触发告警 |
| 前置条件 | 存在监控项，配置retry_times=2 |
| 测试步骤 | 1. 模拟目标服务不可用<br>2. 等待2次探测失败 |
| 预期结果 | 触发告警，发送企业微信通知 |
| 优先级 | P0 |

#### TC-ALERT-002: 恢复通知

| 用例ID | TC-ALERT-002 |
|-------|-------------|
| 用例名称 | 服务恢复发送通知 |
| 前置条件 | 监控项处于warning/critical状态 |
| 测试步骤 | 1. 恢复目标服务<br>2. 等待探测成功 |
| 预期结果 | 发送恢复通知，状态变为normal |
| 优先级 | P0 |

## 9.3 验收标准

### 9.3.1 功能验收

| 验收项 | 验收标准 | 验收方法 |
|-------|---------|---------|
| 用户注册 | 能成功创建账号 | 手工测试 |
| 用户登录 | 能成功登录并获取Token | 手工测试 |
| 创建监控 | 能创建监控项并开始探测 | 手工测试 |
| 探测执行 | 按配置间隔执行探测 | 查看日志 |
| 告警触发 | 连续失败5次触发告警 | 模拟故障 |
| 企业微信通知 | 告警发送到企业微信群 | 查看消息 |
| 恢复通知 | 服务恢复发送通知 | 模拟恢复 |
| 数据隔离 | 用户只能看到自己的数据 | 多账号测试 |

### 9.3.2 性能验收

| 验收项 | 目标值 | 验收方法 |
|-------|:------:|---------|
| 页面加载 | < 2秒 | Lighthouse |
| API响应 | < 500ms | 压测工具 |
| 探测并发 | 100并发 | 监控日志 |
| 系统可用性 | > 99.5% | 监控数据 |
