## 11.1 项目里程碑

### 11.1.1 里程碑规划

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           项目时间线 (8周)                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  W1    W2    W3    W4    W5    W6    W7    W8                               │
│  │     │     │     │     │     │     │     │                                │
│  ▼     ▼     ▼     ▼     ▼     ▼     ▼     ▼                                │
│                                                                              │
│  [需求设计]                                                                  │
│  ├─ 需求文档完成                                                            │
│  └─ 技术方案评审                                                            │

│        ├─ W2: 基础架构 + 用户认证                                            │

│        ├─ W4: 探测引擎 + 告警系统                                            │
│        └─ W5: 状态展示 + 历史记录                                            │
│              │                                                               │

│                    ▼                                                         │

│                    └─ W8: 部署上线 + 验收                                    │

```

### 11.1.2 里程碑详情

| 里程碑 | 时间 | 交付物 | 验收标准 |
|-------|:----:|--------|---------|
| M1: 需求完成 | W1末 | 需求文档 | 文档评审通过 |
| M2: 基础架构 | W2末 | 项目框架 | 能运行Hello World |
| M3: 核心功能 | W5末 | 功能代码 | 单元测试通过 |
| M4: 测试完成 | W7末 | 测试报告 | 测试用例通过率>95% |
| M5: 正式上线 | W8末 | 生产系统 | 验收测试通过 |

## 11.3 风险与应对

| 风险 | 可能性 | 影响 | 应对措施 |
|-----|:------:|:----:|---------|
| 开发延期 | 中 | 高 | 预留缓冲时间，优先级管理 |
| 第三方服务不稳定 | 低 | 高 | 实现重试机制，降级方案 |
| 性能不达标 | 中 | 中 | 提前性能测试，优化方案 |
| 需求变更 | 高 | 中 | 敏捷开发，迭代交付 |
